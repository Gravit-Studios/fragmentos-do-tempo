# Palette System
## Fragmentos do Amanhã / Era Zero

Sistema inicial de paletas para personagens, épocas, UI e elementos temporais.

---

## 1. Objetivo

Garantir que cada época tenha identidade própria sem perder a unidade visual do jogo.

---

## 2. Categorias de paleta

### Paleta global

Usada para elementos recorrentes:

- UI temporal;
- máquina do tempo;
- fragmentos;
- efeitos de anomalia;
- elementos tecnológicos de Theo;
- presença avançada de Voss.

### Paleta por época

Usada para cenário, NPCs e inimigos locais.

### Paleta por personagem

Usada para manter reconhecimento do personagem através das eras.

---

## 3. Cores identitárias

### Tecnologia temporal

- cobre/laranja: tecnologia da máquina, circuitos, relógios, núcleo temporal;
- ciano/azul-esverdeado: energia ativa, lâminas energéticas, interfaces temporais.

### Voss

- preto ou grafite;
- dourado/bronze/cobre;
- branco ou marfim para autoridade ritual;
- vermelho/laranja para acúmulo de poder, quando necessário.

### Naiara

- cor local da época;
- marca herdada em marfim/osso;
- nenhum excesso tecnológico evidente.

### Theo

- base local da época;
- pequeno acento tecnológico;
- cobre/laranja discreto;
- ciano apenas quando o dispositivo temporal estiver ativo.

---

## 5. Regras práticas

- Não usar todas as cores da época no mesmo personagem.
- Cada personagem deve ter uma cor dominante, uma secundária e um acento.
- Efeitos temporais devem se destacar do cenário.
- UI deve permanecer consistente independentemente da época.

---

## 6. Checklist de paleta

Antes de aprovar uma nova arte:

- a época é reconhecível pela cor;
- o personagem ainda é reconhecível;
- o contraste funciona em gameplay;
- não há excesso de cores únicas;
- efeitos temporais não se confundem com magia comum;
- Voss mantém autoridade visual.
